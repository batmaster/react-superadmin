# BooleanInput Component

A versatile boolean input component that supports multiple variants including checkbox, switch, and toggle with comprehensive styling, accessibility features, and form validation integration.

## Overview

The `BooleanInput` component provides a modern, accessible boolean input interface that supports:

- Multiple visual variants (checkbox, switch, toggle)
- Comprehensive accessibility features (ARIA, keyboard navigation)
- Flexible styling and theming with color schemes
- Form validation integration
- Support for indeterminate state (checkbox only)
- Customizable label positioning and sizing

## With Validation and Errors

```tsx live
import React, { useState } from 'react';
import { BooleanInput } from '@react-superadmin/web';

function ValidationExample() {
  const [notifications, setNotifications] = useState(false);
  const [touched, setTouched] = useState(false);

  const hasError = touched && !notifications;
  const errorMessage = hasError
    ? 'You must enable notifications to receive updates'
    : '';

  return (
    <div className='max-w-md space-y-4'>
      <BooleanInput
        label='Enable Notifications'
        variant='switch'
        checked={notifications}
        onChange={value => {
          setNotifications(value);
          setTouched(true);
        }}
        error={errorMessage}
        helperText='Receive important updates and alerts'
        colorScheme='success'
        required
      />

      <button
        onClick={() => setTouched(true)}
        className='px-4 py-2 bg-blue-600 text-white rounded-md hover:bg-blue-700'
      >
        Validate
      </button>
    </div>
  );
}
```

## Indeterminate State (Checkbox Only)

```tsx live
import React, { useState } from 'react';
import { BooleanInput } from '@react-superadmin/web';

function IndeterminateExample() {
  const [items, setItems] = useState([
    { id: 1, name: 'Item 1', checked: false },
    { id: 2, name: 'Item 2', checked: false },
    { id: 3, name: 'Item 3', checked: false },
  ]);

  const allChecked = items.every(item => item.checked);
  const someChecked = items.some(item => item.checked);
  const indeterminate = someChecked && !allChecked;

  const handleSelectAll = (checked: boolean) => {
    setItems(items.map(item => ({ ...item, checked })));
  };

  const handleItemChange = (id: number, checked: boolean) => {
    setItems(items.map(item => (item.id === id ? { ...item, checked } : item)));
  };

  return (
    <div className='max-w-md space-y-4'>
      <BooleanInput
        label='Select All Items'
        variant='checkbox'
        checked={allChecked}
        onChange={handleSelectAll}
        indeterminate={indeterminate}
        helperText='Select or deselect all items'
      />

      <div className='space-y-2'>
        {items.map(item => (
          <BooleanInput
            key={item.id}
            label={item.name}
            variant='checkbox'
            checked={item.checked}
            onChange={checked => handleItemChange(item.id, checked)}
            size='sm'
          />
        ))}
      </div>

      <div className='p-3 bg-gray-50 rounded-md'>
        <p className='text-sm text-gray-600'>
          Selected: {items.filter(item => item.checked).length} of{' '}
          {items.length}
        </p>
      </div>
    </div>
  );
}
```

## Form Integration Example

```tsx live
import React, { useState } from 'react';

    notifications: false,

        label='Auto-save Documents'
        variant='toggle'
        checked={formData.autoSave}
```

## Props Reference

### BooleanInputProps

| Prop             | Type                                                             | Default      | Description                                         |
| ---------------- | ---------------------------------------------------------------- | ------------ | --------------------------------------------------- |
| `checked`        | `boolean`                                                        | `false`      | Current boolean value                               |
| `onChange`       | `(checked: boolean) => void`                                     | **Required** | Callback when value changes                         |
| `variant`        | `"checkbox" \| "switch" \| "toggle"`                             | `"checkbox"` | Visual variant of the input                         |
| `label`          | `string`                                                         | `undefined`  | Label text for the input                            |
| `helperText`     | `string`                                                         | `undefined`  | Helper text below the input                         |
| `error`          | `string`                                                         | `undefined`  | Error message to display                            |
| `required`       | `boolean`                                                        | `false`      | Whether the field is required                       |
| `disabled`       | `boolean`                                                        | `false`      | Whether the input is disabled                       |
| `size`           | `"sm" \| "md" \| "lg"`                                           | `"md"`       | Input size variant                                  |
| `indeterminate`  | `boolean`                                                        | `false`      | Whether to show indeterminate state (checkbox only) |
| `className`      | `string`                                                         | `undefined`  | Custom class names for the container                |
| `inputClassName` | `string`                                                         | `undefined`  | Custom class names for the input element            |
| `labelClassName` | `string`                                                         | `undefined`  | Custom class names for the label                    |
| `colorScheme`    | `"primary" \| "secondary" \| "success" \| "warning" \| "danger"` | `"primary"`  | Color scheme for the input                          |
| `labelPosition`  | `"left" \| "right"`                                              | `"right"`    | Position of the label relative to the input         |

## Variant Details

### Checkbox Variant (`variant="checkbox"`)

- Traditional checkbox input
- Supports indeterminate state
- Accessible with proper ARIA attributes
- Ideal for multiple choice selections
- Works well in forms and lists

### Switch Variant (`variant="switch"`)

- iOS-style toggle switch
- Smooth sliding animation
- Color scheme support
- Perfect for on/off settings
- Modern, intuitive design

### Toggle Variant (`variant="toggle"`)

- Custom toggle button style
- Button-like appearance
- Color scheme support
- Great for feature toggles
- Distinct visual feedback

## Accessibility Features

The BooleanInput component follows accessibility best practices:

- **ARIA Support**: Proper labeling and state attributes
- **Keyboard Navigation**: Full keyboard support for all variants
- **Screen Reader Support**: Clear labels and descriptions
- **Focus Management**: Visible focus indicators
- **Error States**: ARIA invalid attributes for form validation
- **Indeterminate State**: Proper ARIA support for partial selection

## Best Practices

1. **Always provide labels** for screen reader accessibility
2. **Choose appropriate variants** for your use case (checkbox for lists, switch for settings)
3. **Use color schemes consistently** throughout your application
4. **Implement proper validation** with error states when required
5. **Handle loading states** when integrating with data providers
6. **Use indeterminate state** for select-all functionality in lists
7. **Position labels appropriately** for your layout needs
8. **Provide helper text** to guide users on the setting's purpose
